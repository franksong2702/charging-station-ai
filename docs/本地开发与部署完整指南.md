## 🎯 概述

本指南详细说明如何在本地 MacBook 上开发充电桩智能客服系统，同步代码到 GitHub，部署到 Coze 平台，以及查询运行日志。

### 技术栈

- **开发语言**：Python 3.12+
- **工作流框架**：LangGraph 1.0
- **Web 框架**：FastAPI
- **数据库**：PostgreSQL
- **版本控制**：GitHub
- **部署平台**：Coze
- **包管理**：uv

## 💻 本地开发环境搭建

### 前置准备

确保你的 MacBook 已安装：

1. **Homebrew**（包管理器）
```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

2. **Python 3.12+**
```bash
brew install python@3.12
```

3. **Git**
```bash
brew install git
```

4. **PostgreSQL**
```bash
brew install postgresql@16
brew services start postgresql@16
```

### 步骤 1: 克隆代码

```bash
# 在你想存放项目的目录下执行
cd ~/Documents/Projects

# 从 GitHub 克隆
git clone https://github.com/franksong2702/charging-station-ai.git
cd charging-station-ai

# 切换到 dev 分支
git checkout dev
```

### 步骤 2: 安装依赖

项目使用 `uv` 作为包管理器（更现代、更快）：

```bash
# 1. 安装 uv（如果还没有）
curl -LsSf https://astral.sh/uv/install.sh | sh

# 或者用 Homebrew 安装
brew install uv

# 2. 创建虚拟环境并安装依赖
uv sync
```

这会自动：
- 创建 `.venv` 虚拟环境
- 安装 `pyproject.toml` 中的所有依赖

### 步骤 3: 配置本地数据库

```bash
# 1. 创建数据库和用户
createdb charging_station_db
createuser charging_station_user -P
# 输入密码时，设置一个你记得住的密码，比如 "password123"

# 2. 给用户授权
psql -d charging_station_db -c "GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO charging_station_user;"
```

### 步骤 4: 配置环境变量

```bash
# 1. 复制环境变量模板
cp .env.example .env

# 2. 编辑 .env 文件，填入你的配置
# 用你喜欢的编辑器打开，比如：
code .env  # 如果你用 VS Code
# 或者
nano .env
```

在 `.env` 文件中填入以下内容：

```env
# PostgreSQL 数据库配置
DATABASE_URL=postgresql://charging_station_user:password123@localhost:5432/charging_station_db

# 邮件配置（可选，本地开发可以先不填）
EMAIL_RECIPIENT=your-email@example.com
EMAIL_RECIPIENT_NAME=客服团队
```

### 步骤 5: 本地运行项目

#### 方式 1：运行 HTTP 服务（推荐）

```bash
# 启动 FastAPI 服务
uv run python src/main.py -m http -p 5000
```

服务启动后，你可以：
- 访问 **http://localhost:5000/docs** 查看 API 文档（Swagger UI）
- 访问 **http://localhost:5000/health** 检查服务健康状态

#### 方式 2：直接运行工作流测试

```bash
# 测试完整工作流
uv run python src/main.py -m flow -i '{"text": "你好", "session_id": "test123"}'

# 测试单个节点
uv run python src/main.py -m node -n intent_recognition -i '{"text": "充电桩充不进去电"}'
```

### 本地开发常用命令

```bash
# 进入虚拟环境（可选，uv run 会自动用）
source .venv/bin/activate

# 安装新依赖
uv add package-name

# 运行测试（如果有）
uv run pytest

# 代码格式化
uv run black src/

# 启动服务（开发模式，代码变更自动重载）
uv run python src/main.py -m http -p 5000
```

## 🚀 Coze 平台部署

### 前提条件

- ✅ GitHub 仓库已连接到 Coze 平台
- ✅ dev 分支有最新代码
- ✅ 环境变量已在 Coze 平台配置

### 部署步骤

1. **在 Coze 平台界面**：
   - 进入你的项目
   - 点击顶部的 **「部署」** 标签
   - 或者点击右上角的 **「部署」** 按钮
   - 选择要部署的版本（比如 dev 分支的最新提交）
   - 点击部署

2. **Coze 会自动完成**：
   - ✅ 拉取 GitHub 代码
   - ✅ 安装依赖
   - ✅ 配置环境变量（在 Coze 平台的「环境变量」标签配置）
   - ✅ 启动服务
   - ✅ 分配公网访问地址

### 查看部署状态

在 Coze 平台的「部署」页面可以看到：
- **部署版本**：当前部署的 Git 提交
- **部署域名**：你的公网访问地址（如 `wp5bsz5qfm.coze.site`）
- **渠道**：API 已开启
- **环境变量**：已配置的环境变量

## 📊 日志查询

### 方式 1: Coze 平台界面（最简单）

1. 在 Coze 平台点击顶部的 **「日志」** 标签
2. 查看工作流的运行日志
3. 支持搜索和过滤

### 方式 2: 通过 API 查询（推荐）

部署后，可以通过以下 API 查询日志：

#### API 端点

1. **`GET /logs`** - 通用日志查询
2. **`GET /logs/recent`** - 最近日志查询
3. **`GET /logs/search`** - 关键词搜索

#### 使用示例

```bash
# 你的 Coze 部署地址
BASE_URL="https://wp5bsz5qfm.coze.site"

# 1. 获取最近 100 条日志
curl "$BASE_URL/logs?lines=100"

# 2. 获取 ERROR 级别的日志
curl "$BASE_URL/logs?level=ERROR&lines=200"

# 3. 搜索包含"知识库"的日志
curl "$BASE_URL/logs/search?keyword=知识库"

# 4. 获取最近 30 分钟的日志
curl "$BASE_URL/logs/recent?minutes=30"

# 5. 获取最近 1 小时的 ERROR 日志
curl "$BASE_URL/logs/recent?minutes=60&level=ERROR"
```

#### Python 代码调用示例

```python
import requests

# 你的 Coze 部署地址
BASE_URL = "https://wp5bsz5qfm.coze.site"

# 1. 获取最近 100 条日志
response = requests.get(f"{BASE_URL}/logs", params={"lines": 100})
data = response.json()
print(f"共 {data['data']['total']} 条日志")
for log in data['data']['logs']:
    print(log)

# 2. 搜索关键词
response = requests.get(
    f"{BASE_URL}/logs/search",
    params={"keyword": "知识库", "lines": 500}
)
data = response.json()
print(f"找到 {data['data']['total']} 条包含'知识库'的日志")

# 3. 获取最近 1 小时的 ERROR 日志
response = requests.get(
    f"{BASE_URL}/logs/recent",
    params={"minutes": 60, "level": "ERROR"}
)
data = response.json()
print(f"最近 1 小时有 {data['data']['total']} 条 ERROR 日志")
```

#### API 详细文档

##### `GET /logs` - 通用日志查询

**参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `lines` | int | 否 | 要读取的行数，默认 100，最大 1000 |
| `offset` | int | 否 | 偏移量，默认 0 |
| `level` | string | 否 | 日志级别过滤：DEBUG, INFO, WARNING, ERROR |
| `keyword` | string | 否 | 关键词过滤 |
| `start_time` | int | 否 | 开始时间戳（毫秒） |
| `end_time` | int | 否 | 结束时间戳（毫秒） |

**返回示例：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "total": 50,
    "logs": [
      {
        "level": "INFO",
        "message": "Starting run with run_id: abc123",
        "timestamp": "2026-04-03T10:30:00Z"
      }
    ],
    "log_file": "/app/work/logs/bypass/app.log"
  }
}
```

##### `GET /logs/recent` - 最近日志查询

**参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `minutes` | int | 否 | 最近多少分钟，默认 30，最大 1440（24小时） |
| `level` | string | 否 | 日志级别过滤 |
| `keyword` | string | 否 | 关键词过滤 |

##### `GET /logs/search` - 关键词搜索

**参数：**
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `keyword` | string | 是 | 搜索关键词 |
| `lines` | int | 否 | 最多搜索多少行，默认 500，最大 2000 |
| `level` | string | 否 | 日志级别过滤 |

## ❓ 常见问题

### Q: Python 版本不对怎么办？

```bash
# 用 pyenv 管理 Python 版本
brew install pyenv
pyenv install 3.12.0
pyenv local 3.12.0
```

### Q: 数据库连接失败？

- 检查 PostgreSQL 服务是否启动：`brew services list`
- 检查 .env 中的数据库连接字符串是否正确
- 测试连接：`psql postgresql://charging_station_user:password123@localhost:5432/charging_station_db`

### Q: 依赖安装失败？

```bash
# 清理并重新安装
rm -rf .venv
uv sync
```

### Q: 如何查看 Coze 平台的环境变量？

在 Coze 平台点击顶部的 **「环境变量」** 标签，可以查看和配置环境变量。

### Q: 部署后如何验证服务是否正常？

```bash
# 访问健康检查接口
curl "https://wp5bsz5qfm.coze.site/health"

# 查看最近日志
curl "https://wp5bsz5qfm.coze.site/logs/recent?minutes=5"
```
