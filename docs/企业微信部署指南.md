# 充电桩智能客服 - 企业微信部署指南

> **版本**：v1.0  
> **更新日期**：2026-04-03  
> **适用场景**：企业微信自建应用接入充电桩智能客服

---

## 📋 文档概述

本文档指导客户在企业微信中创建自建应用，并接入已部署的腾讯云函数和 Coze API，实现企业微信内的充电桩智能客服功能。

### 前提条件

- ✅ 腾讯云函数已部署完成
- ✅ Coze 平台工作流已部署完成
- ✅ 企业微信管理员权限
- ✅ 企业微信自建应用创建权限

---

## 📖 详细部署步骤

### 第一步：登录企业微信管理后台

1. 访问企业微信管理后台：https://work.weixin.qq.com/
2. 使用企业微信管理员账号扫码登录
3. 进入「应用管理」模块

---

### 第二步：创建自建应用

#### 2.1 进入应用创建页面

1. 在左侧菜单选择「应用管理」
2. 点击「自建」→「创建应用」

#### 2.2 填写应用基本信息

| 配置项 | 配置值 | 说明 |
|--------|--------|------|
| **应用名称** | 充电桩智能客服 | 或自定义名称 |
| **应用Logo** | 上传Logo | 建议尺寸：240×240px |
| **应用介绍** | 为企业员工提供充电桩使用咨询、故障处理等智能客服服务 | 简短描述应用功能 |

#### 2.3 选择应用可见范围

1. 点击「选择部门/成员」
2. 选择可以使用此应用的部门或成员
3. 建议：先选择测试部门，验证通过后再全量开放

---

### 第三步：配置应用信息

#### 3.1 获取应用凭证

创建应用后，在应用详情页可以看到以下重要信息，请**妥善保存**：

| 凭证项 | 位置 | 说明 |
|--------|------|------|
| **AgentId** | 应用详情页顶部 | 应用ID，数字格式 |
| **Secret** | 点击「获取」按钮获取 | 应用密钥，仅显示一次 |
| **CorpId** | 「我的企业」→「企业信息」 | 企业ID，全局唯一 |

⚠️ **重要提醒**：
- Secret 仅显示一次，请立即复制保存
- 如忘记 Secret，可点击「重新获取」，但会导致旧密钥失效

#### 3.2 配置企业微信授权

将以下信息提供给技术方配置腾讯云函数：

```
企业微信配置信息：
├─ CorpId: [你的企业ID]
├─ AgentId: [你的应用ID]
└─ Secret: [你的应用密钥]
```

---

### 第四步：配置接收消息服务器

#### 4.1 获取腾讯云函数地址

技术方提供的腾讯云函数地址格式：

```
https://[区域].apigw.tencentcs.com/release/[函数名]
```

示例：
```
https://ap-guangzhou.apigw.tencentcs.com/release/charging-station-customer-service
```

#### 4.2 配置接收消息

1. 在应用详情页找到「接收消息」模块
2. 点击「设置接收消息」
3. 填写以下信息：

| 配置项 | 配置值 | 说明 |
|--------|--------|------|
| **URL** | [腾讯云函数地址] | 技术方提供的完整URL |
| **Token** | [自定义Token] | 任意字符串，用于验证 |
| **EncodingAESKey** | [随机生成] | 点击「随机生成」按钮 |

#### 4.3 验证URL有效性

1. 填写完上述信息后，点击「保存」
2. 企业微信会发送验证请求到你的腾讯云函数
3. 如配置正确，会显示「验证成功」

⚠️ **注意**：
- 确保腾讯云函数已正确配置企业微信验证逻辑
- 如验证失败，检查腾讯云函数日志排查问题

---

### 第五步：配置应用权限

#### 5.1 配置接口权限

在应用详情页的「权限管理」模块，配置以下权限：

| 权限分类 | 权限项 | 说明 |
|---------|--------|------|
| **基础接口** | 获取访问用户身份 | 用于识别用户身份 |
| **基础接口** | 发送应用消息 | 用于主动推送消息 |
| **通讯录管理** | 读取部门成员（可选） | 如需获取用户信息 |

#### 5.2 配置IP白名单（可选但推荐）

1. 在应用详情页找到「IP白名单」模块
2. 点击「配置」
3. 添加腾讯云函数的出口IP地址

**获取腾讯云函数IP段**：
- 参考腾讯云文档：https://cloud.tencent.com/document/product/583/16038
- 添加对应区域的IP段

---

### 第六步：配置企业微信菜单（可选）

#### 6.1 创建自定义菜单

1. 在应用详情页找到「自定义菜单」模块
2. 点击「设置」
3. 添加菜单项：

**菜单示例**：

```
充电桩客服
├─ 常见问题
│  ├─ 充电操作
│  ├─ 故障处理
│  └─ 费用咨询
├─ 在线客服
│  └─ 联系客服
└─ 关于我们
   └─ 使用帮助
```

#### 6.2 配置菜单跳转

- 菜单类型选择「发送消息」
- 用户点击菜单时，自动发送预设消息给客服

---

### 第七步：测试验证

#### 7.1 内部测试

1. 在企业微信手机端找到「充电桩智能客服」应用
2. 进入应用，发送测试消息：

| 测试项 | 测试消息 | 预期结果 |
|--------|----------|----------|
| 问候语 | `你好` | 收到友好问候回复 |
| 充电咨询 | `第一次用充电桩，怎么操作？` | 收到充电流程说明 |
| 故障处理 | `充不进去电怎么办？` | 收到故障排查步骤 |
| 感谢评价 | `谢谢` | 收到感谢回复和评价选项 |

#### 7.2 兜底流程测试

1. 发送投诉消息：`我要投诉`
2. 确认进入兜底流程
3. 提供测试手机号和车牌号
4. 确认工单创建成功

#### 7.3 多轮对话测试

1. 发送：`特斯拉充电桩怎么扫码？`
2. 收到回复后，发送：`谢谢`
3. 确认显示评价选项
4. 选择评价，确认评价记录保存

---

### 第八步：发布上线

#### 8.1 扩大可见范围

测试通过后，逐步扩大应用可见范围：

1. 第一阶段：测试部门（10-20人）
2. 第二阶段：部分业务部门（100-200人）
3. 第三阶段：全公司开放

#### 8.2 监控和运维

- 关注腾讯云函数监控面板
- 定期查看 Coze 平台运行日志
- 收集用户反馈，持续优化

## 🔧 常见问题排查

### 问题1：应用创建后无法发送消息

**可能原因**：
- 应用可见范围未配置正确
- 接收消息 URL 验证失败

**解决方案**：
1. 检查应用可见范围是否包含测试用户
2. 重新验证接收消息 URL
3. 查看腾讯云函数日志

---

### 问题2：消息发送成功但无回复

**可能原因**：
- 腾讯云函数配置错误
- Coze API 调用失败
- 企业微信凭证不正确

**解决方案**：
1. 检查腾讯云函数环境变量配置
2. 验证 Coze API 密钥是否正确
3. 查看函数执行日志

---

### 问题3：评价或工单数据未保存

**可能原因**：
- 数据库配置错误
- Supabase 连接失败

**解决方案**：
1. 检查数据库连接配置
2. 验证 Supabase API 密钥
3. 查看数据库操作日志
