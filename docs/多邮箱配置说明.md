# 多邮箱配置说明

> **版本**：v1.0  
> **更新日期**：2026-04-03  
> **功能**：支持将工单邮件同时发送到多个邮箱地址

---

## 📋 功能说明

系统现在支持将用户投诉工单邮件同时发送到**多个邮箱地址**，确保多个客服人员都能收到通知。

---

## 🔧 配置方式

### 方式一：环境变量配置（推荐，线上环境）

#### 1. 单个邮箱（向后兼容）

```
EMAIL_RECIPIENT=service1@example.com
EMAIL_RECIPIENT_NAME=客服团队
```

#### 2. 多个邮箱（逗号分隔）

```
EMAIL_RECIPIENT=service1@example.com,service2@example.com,service3@example.com
EMAIL_RECIPIENT_NAME=客服团队
```

#### 3. 第二个邮箱（向后兼容）

```
EMAIL_RECIPIENT=service1@example.com
EMAIL_RECIPIENT_2=service2@example.com
EMAIL_RECIPIENT_NAME=客服团队
```

---

### 方式二：配置文件配置（开发环境）

在 `config/email_config.json` 中配置：

#### 1. 单个邮箱（向后兼容）

```json
{
  "recipient_email": "service1@example.com",
  "recipient_name": "客服团队"
}
```

#### 2. 多个邮箱（推荐）

```json
{
  "recipient_emails": [
    "service1@example.com",
    "service2@example.com",
    "service3@example.com"
  ],
  "recipient_name": "客服团队"
}
```

---

## 📊 配置优先级

系统按以下优先级读取配置：

1. **环境变量**（最高优先级）
   - `EMAIL_RECIPIENT`（支持逗号分隔多个邮箱）
   - `EMAIL_RECIPIENT_2`（第二个邮箱，可选）

2. **配置文件** `config/email_config.json`
   - `recipient_emails`（数组，推荐）
   - `recipient_email`（字符串，向后兼容）

3. **默认值**（最低优先级）
   - `["xuefu.song@qq.com"]`

---

## 💡 使用示例

### 示例 1：两个客服邮箱

**环境变量配置**：
```
EMAIL_RECIPIENT=zhangsan@company.com,lisi@company.com
EMAIL_RECIPIENT_NAME=充电桩客服团队
```

**结果**：工单邮件会同时发送给张三和李四

---

### 示例 2：三个客服邮箱 + 主管邮箱

**环境变量配置**：
```
EMAIL_RECIPIENT=service1@company.com,service2@company.com,service3@company.com,manager@company.com
EMAIL_RECIPIENT_NAME=充电桩客服团队
```

**结果**：工单邮件会同时发送给3个客服和1个主管

---

### 示例 3：使用配置文件

**config/email_config.json**：
```json
{
  "recipient_emails": [
    "service1@company.com",
    "service2@company.com",
    "service3@company.com"
  ],
  "recipient_name": "充电桩客服团队"
}
```

## ⚠️ 注意事项

1. **邮箱格式**：确保每个邮箱地址格式正确
2. **逗号分隔**：多个邮箱用英文逗号 `,` 分隔，不要用中文逗号
3. **空格处理**：系统会自动去除邮箱前后的空格
4. **空邮箱过滤**：空的邮箱地址会被自动过滤
5. **向后兼容**：原有的单邮箱配置仍然有效

## 🎯 快速配置

**最推荐的方式**（环境变量）：

```
EMAIL_RECIPIENT=你的第一个邮箱@example.com,你的第二个邮箱@example.com
EMAIL_RECIPIENT_NAME=充电桩客服团队
```

就这么简单！配置完成后，工单邮件会同时发送到所有配置的邮箱！🎉
