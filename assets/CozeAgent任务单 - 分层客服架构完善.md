## 🎯 二、任务详情

### 任务 1：完善协商处理路由逻辑（30 分钟）🔴

**问题**：当前 `cond_negotiate` 后直接到 `END`，用户拒绝协商方案后无法进入兜底流程。

**需要修改的文件**：`src/graphs/graph.py`

**当前代码**（约第 190 行）：
```python
# ==================== cond_negotiate 分支 → 协商处理路由（后续完善） ====================
# 暂时先让协商处理路由到 END，后续再完善
builder.add_edge("cond_negotiate", END)
```

**修改为**：
```python
# ==================== cond_negotiate 分支 → 协商处理完整路由 ====================
builder.add_conditional_edges(
    source="cond_negotiate",
    path=cond_negotiate_route_path,  # 已导入
    path_map={
        "end": END,  # 用户接受方案 → 结束
        "fallback": "fallback",  # 用户拒绝方案 → 进入兜底
        "negotiate": "negotiate"  # 用户继续追问 → 继续协商
    }
)
```

**验证方法**：
```python
# 测试场景 1：用户接受方案
用户："我要退款"
AI："明白了，请问退款的原因是什么呢？...您看这样可以吗？"
用户："好的" → 应该结束

# 测试场景 2：用户拒绝方案
用户："我要退款"
AI："明白了，请问退款的原因是什么呢？...您看这样可以吗？"
用户："不行，我必须现在退款" → 应该进入 fallback
```

**完成后提交**：
```bash
git add src/graphs/graph.py
git commit -m "feat: 完善协商处理路由 - 支持接受/拒绝/继续三种结果"
```

### 任务 2：执行分层客服专项测试（8 场景）🔴

**测试脚本**：`auto_test_real.py`

**测试场景**：

| ID | 初始消息 | 预期流程 | 验证点 |
|----|---------|---------|--------|
| NS001 | "我要退款" | negotiate | 不直接兜底，先追问原因 |
| NS002 | "优惠券没用上，能退吗" | negotiate | 追问 + 给方案 |
