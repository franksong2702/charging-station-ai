# 腾讯云函数部署指南 - 充电桩智能客服

## 一、需要提供的信息清单

### 1. Coze API信息（必须）

| 信息项 | 获取方式 | 示例值 |
|--------|---------|--------|
| **API访问地址** | Coze部署后的公网地址 | `https://your-domain.com/run` |
| **请求方法** | 固定值 | `POST` |
| **Content-Type** | 固定值 | `application/json` |
| **输入参数** | 见下方 | `user_message` |
| **输出参数** | 见下方 | `reply_content`, `run_id` |

### 2. 请求/响应格式

#### 请求格式
```json
{
  "user_message": "用户发送的消息内容"
}
```

#### 响应格式
```json
{
  "reply_content": "客服回复的内容",
  "run_id": "运行ID"
}
```

## 二、腾讯云函数配置步骤

### 步骤1：创建云函数

1. 登录 [腾讯云函数控制台](https://console.cloud.tencent.com/scf)
2. 点击"新建" → "从头开始"
3. 基础配置：
   - **函数名称**：`charging-station-bot`
   - **运行环境**：Python 3.9
   - **执行方法**：`index.main_handler`
   - **代码上传**：将 `tencent_cloud_function_example.py` 的内容复制进去

### 步骤2：配置环境变量

在"高级配置" → "环境变量"中添加：

| Key | Value | 说明 |
|-----|-------|------|
| `COZE_API_URL` | `https://your-domain.com/run` | Coze工作流API地址 |

### 步骤3：配置触发器

1. 点击"触发器配置" → "创建触发器"
2. 选择"API网关触发器"
3. 配置：
   - **请求方法**：POST
   - **发布环境**：发布
   - **鉴权方法**：免鉴权（或根据需要配置）

### 步骤4：获取访问地址

创建完成后，会生成一个访问地址：
```
https://service-xxx-xxxxxxxx.gz.apigw.tencentcs.com/release/charging-station-bot
```

## 三、企业微信对接配置

### 1. 企业微信配置信息

在腾讯云函数中添加环境变量：

| Key | Value | 说明 |
|-----|-------|------|
| `WECHAT_WEBHOOK_URL` | 企业微信机器人Webhook地址 | 用于发送消息 |
| `COZE_API_URL` | Coze API地址 | 工作流地址 |

### 2. 企业微信回调流程

```
用户发送消息到企业微信
    ↓
企业微信推送到您的回调地址（腾讯云函数）
    ↓
腾讯云函数接收消息
    ↓
调用Coze API处理
    ↓
获取回复内容
    ↓
发送到企业微信
```

## 五、常见问题

### Q1: Coze API地址是什么？

**答案**：Coze工作流部署后会生成一个访问地址，通常是：
- 如果部署在Coze平台：`https://api.coze.cn/v1/workflow/run`（需要token）
- 如果自己部署服务器：`http://your-server-ip:5000/run`

**您需要确认**：
- 您的Coze工作流部署在哪里？
- 公网访问地址是什么？
- 是否需要API Key？

### Q2: 是否需要API Key？

如果您的Coze API需要认证，在云函数中添加：

```python
coze_response = requests.post(
    COZE_API_URL,
    json=coze_payload,
    headers={
        "Content-Type": "application/json",
        "Authorization": f"Bearer {COZE_API_KEY}"  # 添加认证
    },
    timeout=30
)
```

环境变量：
```
COZE_API_KEY=your_api_key_here
```

### Q3: 超时时间设置

腾讯云函数默认超时3秒，建议修改为：
- **执行超时时间**：30秒
- 在"高级配置" → "执行超时时间"中设置

## 七、安全建议

1. **API密钥管理**
   - 不要在代码中硬编码API Key
   - 使用环境变量存储敏感信息
   - 定期更换API Key

2. **访问控制**
   - 配置API网关的IP白名单
   - 或使用企业微信的签名验证

3. **日志监控**
   - 开启云函数日志
   - 监控API调用失败率
   - 设置告警规则
