# 腾讯云函数部署指南

## 📦 部署包内容

```
assets/deploy/
├── cloud_function.py    # 云函数代码
├── serverless.yml       # Serverless配置文件
├── deploy.sh            # 一键部署脚本（推荐）
└── 部署指南.md           # 本文档
```

---

## 🚀 方式一：一键部署（推荐）

### 前置条件

| 工具 | 版本要求 | 安装命令 |
|------|---------|---------|
| Node.js | >= 12.x | `brew install node` (Mac) 或 [官网下载](https://nodejs.org/) |
| npm | >= 6.x | 随Node.js一起安装 |

### 步骤

```bash
# 1. 进入部署目录
cd assets/deploy

# 2. 安装依赖
npm install -g serverless
npm install @serverless/tencent-component --save-dev

# 3. 设置腾讯云密钥（首次需要）
export TENCENT_SECRET_ID=你的SecretId
export TENCENT_SECRET_KEY=你的SecretKey

# 4. 执行部署脚本
chmod +x deploy.sh
./deploy.sh
```

### 获取腾讯云密钥

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/)
2. 点击右上角 -> 访问管理 -> 访问密钥 -> API密钥管理
3. 创建新密钥或使用现有密钥

## 🔧 方式二：控制台手动部署

### 步骤1：创建云函数

1. 登录 [腾讯云函数控制台](https://console.cloud.tencent.com/scf)
2. 点击「新建」->「从头开始」
3. 配置如下：

| 配置项 | 值 |
|--------|-----|
| 函数类型 | 事件函数 |
| 函数名称 | `charging-station-customer-service` |
| 运行环境 | Python 3.9 |
| 执行方法 | `cloud_function.main_handler` |
| 代码提交方式 | 在线编辑 |

4. 将 `cloud_function.py` 的内容复制到编辑器中
5. 点击「完成」

### 步骤2：配置环境变量

在云函数详情页 -> 「函数配置」->「环境变量」，添加：

| 变量名 | 示例值 | 说明 |
|--------|--------|------|
| AI_WORKFLOW_API_URL | `http://1.2.3.4:5000/run` | AI工作流API地址 |
| WECHAT_CORP_ID | `wx1234567890` | 企业微信企业ID |
| WECHAT_AGENT_ID | `100001` | 应用AgentId |
| WECHAT_SECRET | `abc123...` | 应用Secret |
| WECHAT_TOKEN | `mytoken123` | 应用Token |
| WECHAT_ENCODING_AES_KEY | （可选） | 消息加密密钥 |

### 步骤3：配置API网关触发器

1. 在云函数详情页 -> 「触发管理」->「创建触发器」
2. 配置如下：

| 配置项 | 值 |
|--------|-----|
| 触发方式 | API网关触发器 |
| API服务 | 新建API服务 |
| 服务名 | `charging-station-service` |
| 环境 | 发布 |
| 安全类型 | 免鉴权（或自定义） |
| 请求方法 | GET, POST |
| 发布环境 | 发布 |

3. 点击「提交」
4. 记录生成的访问路径（形如：`https://service-xxx.gz.apigw.tencentcs.com/release/`）

### 步骤4：配置企业微信回调

1. 登录 [企业微信管理后台](https://work.weixin.qq.com/)
2. 应用管理 -> 自建应用 -> 选择应用
3. 设置API接收消息

| 配置项 | 值 |
|--------|-----|
| URL | `https://your-api-url/wechat` |
| Token | 与环境变量 `WECHAT_TOKEN` 相同 |
| EncodingAESKey | 与环境变量 `WECHAT_ENCODING_AES_KEY` 相同（可选） |

4. 点击「保存」

## 🔧 常见问题

### Q1: 部署后API调用返回500错误

**解决方案：**
1. 检查环境变量是否正确配置
2. 查看云函数日志，定位具体错误
3. 确认AI工作流API可访问

### Q2: 企业微信验证失败

**解决方案：**
1. 确认Token与企业微信后台配置一致
2. 检查云函数是否正确处理GET请求
3. 查看云函数日志中的验证请求

### Q3: 消息发送失败

**解决方案：**
1. 检查 `WECHAT_CORP_ID`、`WECHAT_AGENT_ID`、`WECHAT_SECRET` 配置
2. 确认企业微信应用已启用
3. 检查用户是否有应用访问权限

### Q4: AI工作流调用超时

**解决方案：**
1. 增加云函数超时时间（默认3秒，建议30秒）
2. 检查AI工作流服务是否正常运行
3. 确认服务器防火墙允许云函数IP访问

## 📊 监控与告警

### 配置告警

在云函数控制台 -> 「监控信息」->「告警配置」：

| 监控指标 | 告警阈值 | 说明 |
|----------|---------|------|
| 调用次数 | > 1000/分钟 | 高并发告警 |
| 错误次数 | > 10/分钟 | 错误告警 |
| 平均耗时 | > 5000ms | 性能告警 |

### 日志持久化

建议开启日志服务（CLS），将云函数日志持久化存储，便于问题排查。
